# skill-harness-grow

## 目的
運用を儀式化させず、各PRでHarness Kitを自然に更新する。

## 入力
- 変更概要（diffまたはPR説明）

## 出力
- S1-S3への影響判定
- 必要なテスト追加/更新
- 必要なDoD/runbook更新
- テンプレに沿ったPR本文

## 手順
1. 変更概要を読む。
2. 聖域への影響を判定する:
   - S1 `{CRITICAL_FLOW_1}`
   - S2 `{CRITICAL_FLOW_2}`
   - S3 `{CRITICAL_FLOW_3}`
3. 影響ありの場合:
   - `doc/evals.md` の対応表にテストを追加/更新。
   - 定義が変わるなら `doc/DoD.md` のガードを更新。
   - 運用/復旧が変わるなら `doc/runbook.md` を更新。
4. PR本文の必須項目を埋める:
   - 影響判定、失敗時挙動、検証、ロールバック。
5. 影響なしの場合:
   - ドキュメントは変更せず、PRに明示的な理由を書く。

## ガードレール
- S1-S3影響が不明のPRはマージしない。
- 更新はコード変更と同じPRで小さく行う。
