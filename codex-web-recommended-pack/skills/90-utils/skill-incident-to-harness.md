# skill-incident-to-harness

## 目的
インシデントの学びを恒久的なHarness更新に変換する。

## 入力
- 症状
- 再現手順
- ログ / 証跡

## 出力
- runbook更新: 症状 -> 確認 -> 対処 -> 恒久対応
- evals更新: 再発防止テスト1本
- 新しい不変条件が見つかった場合はDoDガードを追加

## 手順
1. インシデントの影響範囲とトリガーを要約する。
2. `doc/runbook.md` にエントリを追加する:
   - 症状、確認、応急対応、恒久対応。
3. `doc/evals.md` に再発防止項目を1つ追加する:
   - コマンド/手順と期待結果。
4. インシデントで不変条件の不足が判明した場合:
   - `doc/DoD.md` にガードを追加する。
5. すべての更新をPR本文に関連付ける。

## ガードレール
- ドキュメント更新前にインシデントをクローズしない。
- 記録は短く、実行可能な形にする。
